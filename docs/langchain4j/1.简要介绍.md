## 为什么需要Agent？

### **1. 行政与办公自动化**

- **会议安排 Agent**
  自动读取多人日历空闲时间，协调会议时间，发送邀请，甚至生成会议议程。
- **邮件处理 Agent**
  自动分类、回复常见邮件（如请假审批、客户咨询），标记紧急事项。
- **文档整理 Agent**
  扫描共享盘/邮箱附件，自动归档、命名、提取关键信息（如合同金额、截止日期）。

> ✅ 价值：释放行政人员 30%+ 的事务性时间。

------

### **2. 客户服务与支持**

- **智能客服 Agent**
  7×24 小时回答用户问题（如订单状态、退换货政策），复杂问题转人工。
- **工单分派 Agent**
  根据用户描述自动识别问题类型（技术/账务/物流），分配给对应团队。
- **满意度回访 Agent**
  服务结束后自动发送问卷，分析情感倾向，预警负面反馈。

> ✅ 价值：降低人力成本，提升响应速度与客户体验。

------

### **3. 销售与市场营销**

- **线索挖掘 Agent**
  在 LinkedIn、企业官网、社交媒体上自动识别潜在客户，生成联系人列表。
- **个性化推荐 Agent**
  根据客户历史行为，自动生成定制化产品推荐或营销话术。
- **竞品监控 Agent**
  每日抓取竞品价格、活动、新品发布，生成对比报告。

> ✅ 价值：提高转化率，缩短销售周期。

------

### **4. 研发与工程**

- 代码辅助 Agent（如 GitHub Copilot 的进阶版）不仅补全代码，还能：

  - 自动写单元测试
  - 修复安全漏洞
  - 解释遗留代码逻辑
- **运维监控 Agent**
  实时分析日志，发现异常（如服务器负载突增），自动触发告警或扩容。
- **技术文档生成 Agent**
  根据代码变更自动生成 API 文档或更新说明。
- **自动生成日报周报**
  根据代码产出自动编写报告。

> ✅ 价值：减少重复编码，提升系统稳定性。

------

### **5. 数据分析与决策支持**

- **BI 报表 Agent**
  每日自动生成销售/运营日报，用自然语言总结关键趋势（如“华东区环比下降12%”）。
- **异常检测 Agent**
  监控财务流水、用户行为数据，识别欺诈或异常模式。
- **预测模拟 Agent**
  基于历史数据模拟不同策略效果（如“若降价5%，预计销量增长多少？”）。

> ✅ 价值：让数据驱动决策更实时、更直观。

------

### **6. 人力资源（HR）**

- **简历筛选 Agent**
  根据岗位要求自动匹配候选人，排除明显不符者，标注亮点。
- **面试安排 & 评估 Agent**
  协调面试官与候选人时间，甚至通过视频分析候选人的语言/微表情（需合规）。
- **员工问答 Agent**
  回答关于薪酬、假期、报销流程等 HR 政策问题。

> ⚠️ 注意：涉及隐私和公平性，需谨慎设计。

------

### **7. 跨部门协作增强**

- **项目管理 Agent**
  自动跟踪 Jira/Trello 任务进度，提醒逾期风险，汇总周报。
- **知识库问答 Agent**
  员工输入问题（如“如何申请海外出差？”），Agent 从内部 Wiki 中精准定位答案。
- **多语言翻译 Agent**
  在跨国团队中实时翻译会议记录、文档，消除语言障碍。



## Langchain4j

官网：[LangChain4j 中文文档 | LangChain4j 中文文档](https://docs.langchain4j.info/)

回到Langchain，我们再来看看他到底是个什么玩意，怎么搞出来的Agent，让其能够完成一些任务：

> **LangChain4j = Java 版的 LangChain**
> 它是专为 **Java / Kotlin 开发者**设计的大语言模型（LLM）应用开发框架，让你不用写一堆胶水代码，就能：

- 调用大模型（如通义千问、DeepSeek、Ollama、OpenAI 等）
- 构建 **RAG（检索增强生成）**
- 实现 **Agent（智能体）**
- 做 **对话记忆、工具调用、结构化输出**

✅ **优势**：纯 Java、强类型、与 Spring Boot 无缝集成、支持国产模型！

### 核心组件

| 组件                                    | 作用                                  | 制造业 ERP 场景举例                                     |
| :-------------------------------------- | :------------------------------------ | :------------------------------------------------------ |
| `AiService`                             | 声明式 AI 接口（像写 DAO 一样调 LLM） | `String answer = erpAssistant.ask("上月A产品出货量？")` |
| `RetrievalAugmentor` + `EmbeddingStore` | 实现 RAG                              | 用户问“工单异常原因”，自动查 ERP 日志+知识库再回答      |
| `Tool` / `@Tool`                        | 让 LLM 调用你的 Java 方法             | LLM 自动调用 `getInventory(productId)` 获取库存         |
| `ChatMemory`                            | 保存对话历史                          | 支持多轮追问：“那B产品呢？”                             |
| `OutputParser`                          | 强制 LLM 输出 JSON/POJO               | 返回结构化数据给前端展示                                |

> 可以看到实际逻辑非常简单，就是让llm调用几个预制菜接口完成一些简单任务，比如：
>
> - **分析生产数据**：“上个月注塑车间A班的良品率是多少？”
> - **员工操作手册**：“设备编号 M205 报警了，怎么处理？”（当然优先找人，AI难以担保正确）
> - **自动生成采购/生产建议**：“下季度要不要多备P3001的料？”
> - **异常预警**：“本月注塑能耗超预算 18%，主要因 M203 设备空转时间增加 40 小时”

虽然看起来简单，互联网行业里随便一个实习生就可以完成简单的agent协助。但是一旦涉及到制造业应用了，那在吸引投资，优化生产方面简直就是**夯爆了**！如果不打算去互联网卷生卷死，那实习简历上带个这个玩意再结合一下ERP之类的，hr能直接被唬住，无论如何，**AI应用**这块学习成本低，投资回报概率高，必须学！

### 开始使用

引入：[开始使用 | LangChain4j 中文文档](https://docs.langchain4j.info/get-started)

我们回到正在做的java项目中，加入Langchain4j的pom依赖：

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai</artifactId>
    <version>1.0.0-beta3</version>
</dependency>
```

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j-bom</artifactId>
            <version>1.0.0-beta3</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

> 做好依赖管理，防止后续版本对不上

好的，现在我们就可以开始引入llm模型开始对话了，先用用Langchain提供测试用例：

```java
    @Test
    void contextLoads() {
        OpenAiChatModel model = OpenAiChatModel.builder()
                .baseUrl("http://langchain4j.dev/demo/openai/v1")
                .apiKey("demo")
                .modelName("gpt-4o-mini")
                .build();

        String ans = model.chat("你好");
        System.out.println(ans);
    }
```

### SpringBoot集成

之前我们做的都是直接用Langchain4j提供的接口，其实Spring还额外封装了一套他的Langchain4j：

[Spring Boot 集成 | LangChain4j 中文文档](https://docs.langchain4j.info/tutorials/spring-boot-integration)

> Spring的生态环境真的非常强大！

只需引入以下依赖：

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-spring-boot-starter</artifactId>
    <version>1.0.0-beta3</version>
</dependency>
```

然后到yml配置文件中做配置：

```yml
langchain4j:
  open-ai:
    chat-model:
      # 模型配置
      api-key: demo
      base-url: http://langchain4j.dev/demo/openai/v1
      model-name: gpt-4o-mini
      # 日志
      log-requests: true
      log-responses: true
# 开启系统日志
logging:
  level:
    root: debug
```

此时，我们其实完成的操作是将OpenAiChatModel作为一个Bean注入到了IOC容器中，那么现在我们就可以直接依赖注入调用了：

```java
    @Autowired
    private OpenAiChatModel model;
    
    @Test
    void contextLoads() {
        String ans = model.chat("你好");
        System.out.println(ans);
    }
```
