```yml
      log-requests: true
      log-responses: true
# 开启系统日志

  level:
    root: debug
```



## 本地部署大模型

其实开源的模型我们都可以使用本地资源进行部署，这里我们使用ollama实现本地部署大模型：

### ollama

**ollama是什么？**

> **Ollama = 在你自己的电脑/服务器上，一键运行开源大模型的工具**

- 官网：[https://ollama.com](https://ollama.com/)
- 支持系统：macOS / Windows（WSL2） / Linux
- 核心能力：**本地运行 Llama 3、Qwen、DeepSeek、Phi、Mistral 等主流开源模型**
- 最大优点：**安装简单、命令行操作、API 兼容 OpenAI**

> ✅ 举个例子：
> 你不用申请 API Key，不用联网，只要一条命令：
>
> ```bash
> ollama run qwen:7b
> ```
>
> 就能在自己电脑上跑通义千问 7B 模型，还能对话！
>

注意ollama是无法部署文生图的模型的，需要请使用Stable Diffusion WebUI / ComfyUI这类模型部署助手。

**为什么需要本地大模型？**

| 需求             | Ollama 如何满足                                              |
| :--------------- | :----------------------------------------------------------- |
| **数据不出内网** | 模型和推理全在本地，ERP 数据零外泄                           |
| **无公网依赖**   | 工厂车间没外网？照样跑 AI                                    |
| **成本低**       | 开源模型免费，不用付 Qwen Max 按 token 计费                  |
| **国产化适配**   | 支持 Qwen（通义千问）、DeepSeek（深度求索）等国产模型        |
| **快速迭代**     | 换模型只需改一行配置，比如从 `qwen:7b` → `deepseek-coder:6.7b` |

> 💡 很多制造企业连 GitHub 都访问不了，但 Ollama 可以提前下载好模型离线用！

### 安装ollama

下载网址：[Download Ollama on Windows](https://ollama.com/download/windows)

下载运行exe文件即可

> linux：
>
> ```shell

> ```

### 使用ollama部署一个模型

> 命令行输入`ollama`确认是否完成安装

挑一个喜欢的模型：[Ollama](https://ollama.com/search)

这里选了一个deepseek的r1做样例：

```bash
ollama run deepseek-r1:1.5b
```

这样就可以开始部署了（推荐开个vpn），后续启用这个模型也是这个命令

### ollama命令

> 其实和docker很像，部署的模型就像是容器

| 命令                             | 作用                     | 典型用法示例                                                 | 实用提示                                               |
| -------------------------------- | ------------------------ | ------------------------------------------------------------ | ------------------------------------------------------ |
| `ollama run <model>`             | 启动模型并进入交互式对话 | `ollama run qwen:7b`                                         | 首次运行会自动下载；适合快速测试中文问答               |
| `ollama pull <model>`            | 预下载模型（不启动）     | `ollama pull qwen:1.8b` `ollama pull deepseek-coder:6.7b`    | 推荐加量化后缀如 `-q4_K_M` 节省资源                    |
| `ollama list`                    | 列出本地已安装模型       | `ollama list`                                                | 查看模型名称、大小、修改时间，确认是否下载成功         |
| `ollama show <model>`            | 查看模型详情             | `ollama show qwen:7b --modelfile` `ollama show --parameters` | 可复制 `Modelfile` 用于自定义模型                      |
| `ollama create <name> -f <file>` | 从 Modelfile 创建新模型  | `ollama create erp-qwen -f Modelfile`                        | 关键命令：用于加载从 ModelScope 下载的 `.gguf` 模型    |
| `ollama rm <model>`              | 删除本地模型             | `ollama rm llama3:8b`                                        | 释放磁盘空间（大模型常占 5–10GB）                      |
| `ollama ps`                      | 查看正在运行的模型进程   | `ollama ps`                                                  | 确认 Ollama 后台服务是否正常（LangChain4j 依赖此服务） |
| `ollama serve`                   | 手动启动 API 服务        | （一般无需执行）                                             | 默认安装后自动运行，API 地址：`http://localhost:11434` |
| `ollama help`                    | 查看帮助                 | `ollama help`                                                | 快速查阅所有子命令                                     |

### 在项目中调用ollama部署的本地模型

Langchain4j同样也封装了一套针对ollama的接口，引入依赖即可：[Ollama | LangChain4j 中文文档](https://docs.langchain4j.info/integrations/language-models/ollama)（上面有的加载不出来，向下翻）

对于SpringBoot项目，我们只需要引入SpringBoot提供的组合starter即可：

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-ollama-spring-boot-starter</artifactId>
</dependency>
```

> 做了**版本管理**就不用加version了：
>
> ```xml
> <dependencyManagement>
>     <dependencies>
>         <dependency>
>             <groupId>dev.langchain4j</groupId>
>             <artifactId>langchain4j-bom</artifactId>
>             <version>1.0.0-beta3</version>
>             <type>pom</type>
>             <scope>import</scope>
>         </dependency>
>     </dependencies>
> </dependencyManagement>
> ```

同样的，接下来我们要准备好Bean的相关信息，写在yml中即可：

```yml
langchain4j:
  open-ai:
    chat-model:
      # 模型配置
      api-key: ${DEEPSEEK_APIKEY}
      base-url: https://api.deepseek.com
      model-name: deepseek-chat

  ollama:
    chat-model:
      base-url: http://localhost:11434
      model-name: deepseek-r1:1.5b
```

可以看到配置和外部调用的openai基本大差不差，只是不需要apikey了，相关参数参见：[Ollama | LangChain4j 中文文档](https://docs.langchain4j.info/integrations/language-models/ollama)中的**参数部分**

> 这里注意：不管是本机还是其他服务器，**只要部署的方式是ollama，就请使用这一套ollama的接口！**

```java
@Autowired
private OllamaChatModel model;

@Test
void contextLoads() {
    String ans = model.chat("你是谁？");
    System.out.println(ans);
}
```

> 注意使用**OllamaChatModel**作为注入类型，**ChatLanguageModel**被**OpenAiChatModel**与**OllamaChatModel**同时实现，@Autowired无法注入同时配置了两种实现类的Bean！

## 接入阿里百炼

进入控制台[大模型服务平台百炼控制台](https://bailian.console.aliyun.com/cn-beijing/?tab=model#/api-key)注册完成账号后创建自己的key

### 接入项目

首先我们先到部署项目的系统中载入好系统变量，项目中直接$读取即可

> IDEA进行测试/运行的时候记得关一下IDEA再开，不然不会加载

然后我们到项目中引入好依赖：参考[DashScope (通义千问) | LangChain4j 中文文档](https://docs.langchain4j.info/integrations/language-models/dashscope)

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-community-dashscope-spring-boot-starter</artifactId>
</dependency>

<!-- 依赖管理 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j-community-bom</artifactId>
            <version>${langchain4j.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

> 阿里的一套普遍需要额外的依赖管理，推荐使用时加上

然后就开始配置使用信息：

```yml
langchain4j:
  community:
    dashscope:
      chat-model:
        api-key: ${ALIBAILIAN_APIKEY}
        model-name: qwen-plus
```

> 不用填url是因为使用依赖注入的Bean对象内部已经封装了

那么现在就可以开始测试了：

```java
@Autowired
private QwenChatModel model;

@Test
void contextLoads() {

}
```

### 使用文生图

说明文档：[大模型服务平台百炼控制台](https://bailian.console.aliyun.com/cn-beijing/?tab=api#/api/?type=model&url=2975126)

```java
@Test
void test2() {
    // 修复1: 添加空值检查和默认值
    String apiKey = System.getenv("ALIBAILIAN_APIKEY");
    if (apiKey == null || apiKey.isEmpty()) {
        throw new IllegalStateException("请设置环境变量 ALIBAILIAN_APIKEY");
    }

    // 修复2: 使用正确的模型名称和添加必要配置
    WanxImageModel model = WanxImageModel.builder()
        .modelName("wanx-v1") // 修正模型名称
        .apiKey(apiKey)
        .build();

    try {
        Response<Image> response = model.generate("看起来就强大的骑士");
        System.out.println("生成的图片URL: " + response.content().url());
    } catch (Exception e) {
        System.err.println("图片生成失败: " + e.getMessage());
        e.printStackTrace();
    }
}
```

> 此为**旧版调用方式**，**新版模型无法使用**，推荐还是使用目前说明文档使用的调用方式！
> 但注意版本对应，有可能需要的旧版不支持新版调用方式！

最终的返回结果为生成的图片url而不是直接返回图片，需要可根据url进行下载以及展示
