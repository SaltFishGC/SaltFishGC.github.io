GEO数据结构

除了先前的一些基本数据类型，Redis引入了一些额外的数据类型，地理坐标geo就在其中：

  • Redis 使用 Sorted Set 存储地理位置：
    • 成员（member）：位置名称（如 "user:123"、"shop:beijing"）
    • score：该位置的 52 位整数编码（通过 GeoHash 算法 将经纬度转换而来）
命令功能语法典型参数/选项返回值注意事项
GEOADD添加一个或多个地理位置GEOADD key longitude latitude member [longitude latitude member ...]- 经度范围：-180 ～ +180 - 纬度范围：-85.0511 ～ +85.0511成功添加的成员数量（整数）• 同名 member 会覆盖旧位置 • 底层是 ZADD
GEOPOS获取指定成员的经纬度GEOPOS key member [member ...]支持批量查询每个成员对应 [longitude, latitude]，不存在则返回 nil返回高精度浮点数
GEODIST计算两个位置间的距离GEODIST key member1 member2 [unit]unit 可选： m（米，默认）、km、ft、mi距离数值（字符串），单位由参数决定；成员不存在返回 nil基于 Haversine 公式（球面距离）
GEORADIUS （已废弃）根据坐标搜索附近位置`GEORADIUS key longitude latitude radius unit [WITHCOORD] [WITHDIST] [WITHHASH] [COUNT n] [ASCDESC] [STORE key]`• radius：半径 • unit：m/km/ft/mi • WITHxxx：返回附加信息 • COUNT：限制结果数成员列表（可带距离、坐标等）
GEORADIUSBYMEMBER （已废弃）根据已有成员搜索附近位置GEORADIUSBYMEMBER key member radius unit [选项同上]同上同上⚠️ Redis 6.2+ 已废弃
GEOSEARCH （推荐 ✅）统一地理搜索命令（坐标 or 成员）`GEOSEARCH key FROM{LONLATMEMBER} ... BY{RADIUSBOX} ... [WITHDIST] [WITHCOORD] [COUNT n] [ASCDESC]`
ZREM删除某个地理位置ZREM key member [member ...]—成功删除的数量GEO 无专用删除命令，因底层是 ZSet

🔔 补充说明：

  • 底层存储：所有 GEO 数据实际存储在 Sorted Set 中，可用 ZRANGE key 0 -1 WITHSCORES 查看 GeoHash 值。
  • 精度：约 0.5 米（52 位 GeoHash 编码）。
  • 性能：适合 百万级以内 的实时 LBS 场景；超大规模建议结合 Elasticsearch 或专业 GIS。
  • 单位注意：GEODIST 和 GEOSEARCH 中的单位必须显式指定（如 km），否则默认为 米（m）。
特性说明
底层实现Sorted Set + GeoHash
核心能力存储位置、算距离、搜附近
适用场景LBS、附近推荐、实时定位
替代方案大数据量用 ES / PostGIS；高并发用 Redis GEO
注意Redis 6.2+ 优先用 GEOSEARCH

💡 一句话记住： Redis GEO = 用 Sorted Set 存位置，用 GeoHash 快速筛，用 Haversine 精确算。

附件商铺搜索

按照如上给出的Redis的geo数据类型提供的功能，我们可以先存储商家经纬度到Redis的Geo zset中，再使用geosearch搜索附近的店铺。

同时，推荐商店一般会推荐类似的商店，我们这里简单使用数据库的type字段对商店类型做了一个分类（考虑使用Enums维护类型与代号映射），推荐时就是用这个类型来做同类推荐。而到Redis中，我们选择将所有同一type的商店存至同一个Geo zset的value中，使用type拼接为key。

    @Override
    public Result queryShopByType(Integer typeId, Integer current, Double x, Double y) {
        // 坐标可能为null，直接分页即可
        if (x == null || y == null) {
            // 根据类型分页查询
            Page<Shop> page = query()
                    .eq("type_id", typeId)
                    .page(new Page<>(current, SystemConstants.DEFAULT_PAGE_SIZE));
            // 返回数据
            return Result.ok(page.getRecords());
        }
        // 根据类型分页查询
        int from = (current - 1) * SystemConstants.DEFAULT_PAGE_SIZE;   // 计算分页开始索引
        int end = current * SystemConstants.DEFAULT_PAGE_SIZE;  // 计算分页结束索引

        // 获取附件相似类型店铺，typeid为key，输入xy以及范围，个数，分页
        String key = RedisConstants.SHOP_GEO_KEY + typeId;
        GeoResults<RedisGeoCommands.GeoLocation<String>> search = stringRedisTemplate.opsForGeo().search(
                key,                                // key
                GeoReference.fromCoordinate(x, y),  // 输入坐标
                new Distance(50),             		// 范围
                RedisGeoCommands.GeoSearchCommandArgs.newGeoSearchArgs()
                        .includeDistance()
                        .limit(end)                 // 查询个数，from无法指定，只能从0开始返回结果，需要自行处理0~end
        );
        // 获取店铺id
        if (search == null || search.getContent().isEmpty()) {   // 没有结果
            return Result.ok();
        }
        List<GeoResult<RedisGeoCommands.GeoLocation<String>>> content = search.getContent();
        
        if (content.size() <= from){    // 分页后没有结果
            return Result.ok();
        }
        // 对数据做处理
        // 截取from ~ end
        List<Long> ids = new ArrayList<>(content.size());
        Map<String, Distance> map = new HashMap<>(content.size());
        content.stream().
                skip(from).                                         // 跳过from个数
                forEach(item -> {
                    String name = item.getContent().getName();
                    ids.add(Long.valueOf(name));
                    Distance distance = item.getDistance();
                    map.put(name, distance);
                });

        // 根据id查询
//        List<Shop> list = query().in("id", ids).last("ORDER BY FIELD(id," + ids + ")").list();
        String idsStr = StrUtil.join(",", ids);
        List<Shop> list = list(new QueryWrapper<Shop>().
                in("id", ids).
                last( "ORDER BY FIELD(id," + idsStr + ")")
        );
        for (Shop shop : list) {
            shop.setDistance(map.get(shop.getId().toString()).getValue());
        }

        return Result.ok(list);
    }
