用户签到

SaltFishGC2025-1-23

我们虽然可以使用数据库来实现对用户签到数据的记录，但是这样一来数据量就会达到恐怖的量级，那么我们是否可以使用其他方式来存储用户的签到情况呢？

假如我们用0来代表未签到，1代表签到，那么30/31位bit，4个字节就可以记录某一用户某一月的签到情况，而再多几位来表示年份以及月份即可表示实现记录该年中该月的签到情况

Redis中的BitMap

Redis 中的 Bitmap（位图） 并不是一个独立的数据类型，而是基于 String（字符串） 类型实现的一种高效存储布尔值（0/1） 的数据结构。它利用每个字节的 8 个 bit 位来表示状态，极大节省内存，非常适合处理海量二值状态场景。表格

命令功能示例
SETBIT key offset value设置某一位的值（0 或 1）SETBIT user:sign:20240601 123 1
GETBIT key offset value获取某一位的值GETBIT user:sign:20240601 123 → 返回 1
BITCOUNT key [start end]统计 1 的个数（可选字节范围）BITCOUNT user:sign:20240601
BITPOS key bit [start end]查找第一个指定 bit（0 或 1）的位置BITPOS user:sign:20240601 0 → 找第一个未签到用户
BITOP operation destkey key1 key2 ...对多个 Bitmap 做位运算（AND/OR/XOR/NOT）BITOP AND result k1 k2
BITFIELD key [GET type offset] [SET type offset value] [INCRBY type offset increment] [OVERFLOW WRAP SAT FAIL]获取/设置某一段的值BITFIELD user:1001 SET u2 9 0

应用场景：

✅ 1. 用户签到 / 打卡系统

  • 每天一个 key：sign:20240601
  • 用户 ID 作为 offset
  • 签到：SETBIT sign:20240601 1001 1
  • 查是否签到：GETBIT sign:20240601 1001
  • 统计当日签到人数：BITCOUNT sign:20240601

💡 内存对比：

  • 100 万用户签到：
    • 用 Set 存用户 ID → 约 50～100 MB
    • 用 Bitmap → 仅需 1000000 / 8 / 1024 ≈ 122 KB

✅ 2. 活跃用户统计（DAU/MAU）

  • 每天生成一个 Bitmap（记录当天活跃用户）
  • 计算周活：对 7 天 Bitmap 做 BITOP OR week_active day1 day2 ... day7
  • BITCOUNT week_active → 周活人数

🔍 支持任意时间段组合（如“工作日活跃”、“连续活跃”等）

✅ 3. 布隆过滤器（Bloom Filter）底层实现

  • Bloom Filter 需要多个 hash 函数映射到 bit 位
  • Redis Bitmap 可作为其存储后端（但官方推荐用 RedisBloom 模块）

✅ 4. 权限控制 / 标签系统

  • 每个权限对应一个 bit 位
  • 用户权限 = 一个 Bitmap
  • 判断是否有权限 A（第 3 位）：GETBIT user:perm:1001 3

在业务中借助bitmap实现签到

签到有两种做法，如果人数可控，且更加需要检查员工/用户整体签到情况，那么我们可以将key设置为yyyyMMdd表示每一天，而内部的value bitmap的每一位表示对应编号员工/用户的签到情况。

如果人数比较多，且业务更加注重与每一位用户自己的签到情况，那么我们可以将key设置为yyyyMM + userid来表示某一用户的一个月，内部value bitmap的每一位表示该用户该月份的签到情况。

Redis的bitmap自动以字节为单位扩容，当我们set一个key的索引为6的点位为1/0时，会对从前往后数7号位置操作（索引从0开始，lua从1开始），这时会自动申请一个字节（8bit）的空间用于存储该bitmap；若是操作第12号点位，则会扩容至两个字节以存储该bitmap；

所以不必担心其占用空间不好控制的问题，其的申请空间是动态的（尽可能小）。

而目前的业务更倾向于统计每个用户自己的签到情况，那么我们就将key设置为 suffix + userid + yyyyMM 的形式，value的bitmap用32位bit存储该月的签到情况：

@Override
public Result sign() {
    UserDTO user = UserHolder.getUser();
    String key = RedisConstants.USER_SIGN_KEY + user.getId();
    // 获取当前时间，转化为日期
    LocalDateTime now = LocalDateTime.now();
    String suffix = now.format(DateTimeFormatter.ofPattern(":yyyyMM"));
    key = key + suffix;

    // 获取当前在月份中的第几天
    int day = now.getDayOfMonth();

    // 设置用户签到，注意索引从0开始 setbit key offset value
    stringRedisTemplate.opsForValue().setBit(key, day - 1, true);

    return Result.ok();
}

签到统计

**需求：**统计本月连续签到天数

我们可以借助Redis的bitmap提供的统计接口，逆序统计某一索引向前的连续值为0/1的次数，当获取到未签到的标识时停止计数：

@Override
public Result signCount() {
    UserDTO user = UserHolder.getUser();
    String key = RedisConstants.USER_SIGN_KEY + user.getId();
    // 获取当前时间，转化为日期
    LocalDateTime now = LocalDateTime.now();
    String suffix = now.format(DateTimeFormatter.ofPattern(":yyyyMM"));
    key = key + suffix;

    // 获取当前在月份中的第几天
    int day = now.getDayOfMonth();

    // 获取本月的签到记录 bitfield key get i
    List<Long> result = stringRedisTemplate.opsForValue().bitField(
            key,
            BitFieldSubCommands.create().
                    get(BitFieldSubCommands.BitFieldType.unsigned(day)).valueAt(0)
    );

    // 判断结果
    if (result == null || result.isEmpty()){
        return Result.ok(0);
    }
    Long num = result.get(0);
    if (num == null || num == 0){
        return Result.ok(0);
    }

    // 循环遍历
    int count = 0;
    // 与运算，和1与运算后，前多个bit位都被置0，只余最后一个bit位，为1，则count加1
    while ((num & 1) != 0){
        count++;
        // >> 为带符号右移（算术右移）高位补符号位
        // >>> 无符号右移（逻辑右移）高位补0
        num = num >>> 1;
    }

    return Result.ok(count);
}

List<Long> result = stringRedisTemplate.opsForValue().bitField(
    key,
    BitFieldSubCommands.create().
        get(BitFieldSubCommands.BitFieldType.unsigned(day)).valueAt(0)
);

返回值类型 返回类型: List<Long> 实际内容: 包含一个 Long 元素的列表，代表指定位置的位域值

命令解析BitFieldSubCommands.create() 创建一个 Redis BitField 命令构建器

get(BitFieldType.unsigned(day)) unsigned(day): 指定获取无符号整数，宽度为 day 位 day 来自 now.getDayOfMonth()，表示当前月份的天数 例如：如果是15号，则获取15位的无符号整数

valueAt(0) 从偏移量0开始读取位值 从最左边的位开始获取指定数量的位，即从0~输入day日期-1

实际作用 获取本月签到记录: 从Redis中获取存储的本月签到bitmap 位图解码: 将连续的签到位转换为一个整数 连续签到计算: 通过 (num & 1) 和 num >>> 1 操作逐位检查连续签到情况

假如需要统计跨月连续签到呢？

其实也可以记录上次签到时间和连续签到次数，假如签到后检测到上次签到时间是前一天，那么就次数+1同时刷新上次签到时间；假如检测到上次签到时间不是上一天，那么次数归零/变为1。（今日签过不能再签）
