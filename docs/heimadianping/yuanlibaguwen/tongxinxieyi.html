Redis通信协议

RESP协议

RESP（REdis Serialization Protocol） 是 Redis 客户端与服务端之间通信的协议，设计目标是：简单、快速解析、可读性强、支持多种数据类型。

特性说明
基于文本可读（调试方便），但非纯文本（有二进制安全支持）
请求-响应模型客户端发送命令，服务端返回结果
支持多种数据类型简单字符串、错误、整数、批量字符串、数组
二进制安全能传输任意字节（包括 \0）
无状态每条命令独立解析

💡 RESP 是 Redis 高性能的关键之一：解析开销极低，C 语言可逐字节处理，无需复杂状态机。

5种数据类型：

每种类型以首字节标识符开头：

类型标识符说明示例
简单字符串（Simple String）+非二进制安全，不含换行+OK\r\n
错误（Error）-服务端错误信息-ERR unknown command\r\n
整数（Integer）:64 位有符号整数:1000\r\n
批量字符串（Bulk String）$二进制安全，可含 \0 $ 5\r\nhello\r\n
数组（Array）*元素可以是任意 RESP 类型*2\r\n $ 3\r\nGET\r\n $ 5\r\nmykey\r\n

📌 所有类型都以 \r\n（CRLF）结尾。
