## 负载均衡流程

我们知道微服务间远程调用都是有OpenFeign帮我们完成的，甚至帮我们实现了服务列表之间的负载均衡。但具体负载均衡的规则是什么呢？何时做的负载均衡呢？

在SpringCloud的早期版本中，负载均衡都是有Netflix公司开源的**Ribbon**组件来实现的，甚至Ribbon被直接集成到了Eureka-client和Nacos-Discovery中。

但是自SpringCloud2020版本开始，已经弃用Ribbon，改用Spring自己开源的**Spring Cloud LoadBalancer**了，我们使用的OpenFeign的也已经与其整合。

| Spring Cloud 版本 | 默认负载均衡策略                  |
| :---------------- | :-------------------------------- |
| 2020.x 之前       | Ribbon（轮询）                    |
| 2020.x 及之后     | Spring Cloud LoadBalancer（轮询） |

```
┌─────────────────────────────────────────────────────────────────┐
│              Spring Cloud LoadBalancer 架构                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────┐                                                │
│  │  OpenFeign  │                                                │
│  └──────┬──────┘                                                │
│         ↓                                                       │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │           Spring Cloud LoadBalancer                     │    │
│  │  ┌───────────────┐    ┌───────────────────────────────┐ │    │
│  │  │ LoadBalancer  │ →  │ ServiceInstanceListSupplier   │ │    │
│  │  │ (选择策略)     │    │ (从注册中心获取实例列表)          │ │    │
│  │  └───────────────┘    └───────────────────────────────┘ │    │
│  └─────────────────────────────────────────────────────────┘    │
│         ↓                                                       │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐          │
│  │  Nacos      │    │  Eureka     │    │  Consul     │          │
│  │  注册中心    │    │  注册中心     │    │  注册中心    │          │
│  └─────────────┘    └─────────────┘    └─────────────┘          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

具体的**调用流程**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    完整调用流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 服务启动                                                     │
│     ↓                                                           │
│     注册到注册中心 (Nacos/Eureka)                                 │
│     实例信息：{host, port, metadata, weight}                     │
│                                                                │
│  2. 调用方发起请求                                                │
│     ↓                                                           │
│     @FeignClient(name = "user-service")                         │
│                                                                 │
│  3. LoadBalancer 介入                                            │
│     ↓                                                           │
│     a. 从缓存/注册中心获取 user-service 实例列表                    │
│     b. 根据策略选择一个实例                                        │
│     c. 返回 ServiceInstance                                     │
│                                                                 │
│  4. 发起实际 HTTP 请求                                            │
│     ↓                                                           │
│     http://selected-host:port/api/xxx                           │
│                                                                 │
│  5. 请求完成，更新统计信息（如响应时间）                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```



## 切换负载均衡算法

之前分析源码的时候我们发现负载均衡的算法是有`ReactiveLoadBalancer`来定义的，我们发现它的实现类有三个：

- **RoundRobinLoadBalancer**（轮询）
- **RandomLoadBalancer**（随机）
- **NacosLoadBalancer**（Nacos配置）

其中`RoundRobinLoadBalancer`和`RandomLoadBalancer`是由`Spring-Cloud-Loadbalancer`模块提供的，而`NacosLoadBalancer`则是由`Nacos-Discorvery`模块提供的。

```java
public class OpenFeignConfig {

    @Bean
    public ReactorLoadBalancer<ServiceInstance> reactorServiceInstanceLoadBalancer(
            Environment environment, 
        	NacosDiscoveryProperties properties,
            LoadBalancerClientFactory loadBalancerClientFactory) {
        String name = environment.getProperty(LoadBalancerClientFactory.PROPERTY_NAME);
        return new NacosLoadBalancer(
                loadBalancerClientFactory.getLazyProvider(
                    name, 
                    ServiceInstanceListSupplier.class), 
            name, 
            properties);
    }
}
```

> 不要在这里加`@Configuration`注解，要让其针对client装配

由于这个OpenFeignConfig没有加`@Configuration`注解，也就没有被Spring加载，因此是不会生效的。接下来，我们要在**启动类**上通过注解来声明这个配置。

有两种做法：

- 全局配置：对所有服务生效

```Java
@LoadBalancerClients(defaultConfiguration = OpenFeignConfig.class)
```

- 局部配置：只对某个服务生效

```Java
@LoadBalancerClients({
        @LoadBalancerClient(value = "item-service", configuration = OpenFeignConfig.class)
})
```

> 如果只需要切换到随机，其实也可以在yml配置中切换：
>
> ```yml
> # application.yml
> 
> # 全局负载均衡策略配置
> spring:
>   cloud:
>     loadbalancer:
>       # 方式1：指定策略名称（2022.x+ 版本）
>       configurations: random  # 可选值：round-robin, random
>       
>       # 方式2：缓存配置
>       cache:
>         enabled: true
>         ttl: 35s
>         capacity: 256
> ```

Nacos的负载均衡其实就是优先本集群然后**加权随机**，对于权值，我们直接到nacos可视化界面设置对应服务的集群实例即可配置权重。

> 权重也可以直接在yml中配置：
>
> ```yml
> # order-service (提供者)
> spring:
>   application:
>     name: order-service
>   cloud:
>     nacos:
>       discovery:
>         server-addr: localhost:8848
>         metadata:
>           weight: 10  # 权重配置
>           version: 2.0.0
> ```
