条件构造器Wrapper

SaltFishGC2026-1-29

MyBatis-Plus（MP）的 条件构造器（Wrapper） 是其核心亮点之一，用于构建复杂的查询、更新条件，无需手写 SQL 字符串，避免拼接错误，同时支持链式调用、Lambda 表达式，类型安全且可读性强。

一、主要类型

类型用途
QueryWrapper<T>查询条件（SELECT）
UpdateWrapper<T>更新条件（UPDATE）
LambdaQueryWrapper<T>查询条件 + Lambda 表达式（推荐）
LambdaUpdateWrapper<T>更新条件 + Lambda 表达式（推荐）

✅ 优先使用 LambdaQueryWrapper / LambdaUpdateWrapper：字段名通过方法引用（如 User::getName）获取，避免硬编码字符串，重构安全。

二、基本用法示例

1. 查询：LambdaQueryWrapper

@Autowired
private UserMapper userMapper;

// 查询：name 以 "张" 开头 且 age >= 18 且 email 不为空
List<User> list = userMapper.selectList(
    new LambdaQueryWrapper<User>()
        .likeRight(User::getName, "张")     // name LIKE '张%'
        .ge(User::getAge, 18)              // age >= 18
        .isNotNull(User::getEmail)         // email IS NOT NULL
);

2. 更新：LambdaUpdateWrapper

// 将所有年龄 < 18 的用户 status 设为 0
userMapper.update(
    new User().setStatus(0), // SET status = 0
    new LambdaUpdateWrapper<User>()
        .lt(User::getAge, 18) // WHERE age < 18
);

注意：update() 第一个参数是 要更新的实体对象（SET 部分），第二个是条件（WHERE 部分）。

三、常用条件方法（通用）

方法SQL 效果示例
eq=.eq(User::getAge, 20) → age = 20
ne!=.ne(User::getName, "张三")

⚠️ last() 有 SQL 注入风险，仅用于无法用 Wrapper 表达的场景。

四、高级用法

1. 条件嵌套（AND / OR）

// (name = '张三' AND age > 20) OR (email LIKE '%@qq.com')
new LambdaQueryWrapper<User>()
    .and(i -> i.eq(User::getName, "张三").gt(User::getAge, 20))
    .or()
    .like(User::getEmail, "@qq.com");

2. 动态条件（if 判断）

String name = "张";
Integer minAge = null;

LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.likeRight(StringUtils.isNotBlank(name), User::getName, name)
       .ge(minAge != null, User::getAge, minAge); // 自动跳过 null 条件

所有条件方法都支持 第一个参数为 boolean，用于控制是否拼接该条件。

3. SELECT 指定字段（避免查全表）

// 只查 id 和 name
List<User> list = userMapper.selectList(
    new LambdaQueryWrapper<User>()
        .select(User::getId, User::getName)
        .eq(User::getStatus, 1)
);

4. 子查询

// WHERE age IN (SELECT age FROM user WHERE dept_id = 10)
new LambdaQueryWrapper<User>()
    .in(User::getAge, 
        new LambdaQueryWrapper<User>()
            .select(User::getAge)
            .eq(User::getDeptId, 10)
    );

五、UpdateWrapper 特有功能

除了 WHERE 条件，UpdateWrapper 还可以直接在条件中设置字段值（无需传实体）：

// 等价于：UPDATE user SET age = age + 1 WHERE id = 1
userMapper.update(null,
    new LambdaUpdateWrapper<User>()
        .setSql("age = age + 1") // ❌ 不推荐，有注入风险
        .eq(User::getId, 1)
);

// ✅ 推荐：用 set() 方法
userMapper.update(null,
    new LambdaUpdateWrapper<User>()
        .set(User::getAge, 25) // SET age = 25
        .eq(User::getId, 1)
);

如果同时传了实体和 set()，实体字段优先级更高。

六、注意事项

  1. 字段名必须对应数据库列名（或通过 @TableField 映射）。
  2. Lambda 表达式要求字段有 getter 方法（Lombok 的 @Data 可自动生成）。
  3. 不要滥用 last()，尽量用 Wrapper 提供的方法。
  4. 复杂逻辑仍可回退到 XML：Wrapper 不是万能的，多表 JOIN、复杂聚合建议用原生 MyBatis。

总结

场景推荐写法
简单单表查询/更新LambdaQueryWrapper / LambdaUpdateWrapper
避免字段名硬编码用 Entity::getField（Lambda）
动态条件利用 condition, column, value 三参数形式
复杂 SQL混合使用 XML + Wrapper

💡 记住口诀： “单表操作用 Wrapper，多表关联写 XML；Lambda 防错又安全，动态条件靠 boolean。”

通过条件构造器，MyBatis-Plus 让 80% 的 CRUD 场景告别手写 SQL，大幅提升开发效率与代码健壮性。
