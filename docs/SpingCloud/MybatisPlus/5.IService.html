IService

SaltFishGC2026-1-29

IService 是 MyBatis-Plus（MP） 提供的一个通用 Service 层接口，它封装了大量常用的 CRUD 操作（比 BaseMapper 更高层），让你在 Service 层几乎不用写重复代码，大幅提升开发效率。

一、核心定位

层级MP 提供的抽象
DAO/Mapper 层BaseMapper<T>（17+ 个方法）
Service 层IService<T>（60+ 个方法，含批量、链式、分页、Lambda 等）

✅ IService 是对 BaseMapper 的进一步封装，内置事务、批处理优化、链式调用支持。

二、基本使用方式

1. 自定义 Service 接口（可选）

public interface UserService extends IService<User> {
    // 可以扩展自定义方法
    List<User> getActiveUsers();
}

2. Service 实现类继承 ServiceImpl

@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {

    @Override
    public List<User> getActiveUsers() {
        return this.list(new LambdaQueryWrapper<User>().eq(User::getStatus, 1));
    }
}

🔸 泛型 <UserMapper, User>：

  • 第一个：你的 Mapper 接口
  • 第二个：实体类

三、IService 提供的核心能力（常用方法）

✅ 1. 基础 CRUD（比 Mapper 更语义化）

// 插入
boolean save(User entity);                // 单条
boolean saveBatch(Collection<User> list); // 批量（默认 1000 条一批）

// 删除
boolean removeById(1L);
boolean removeByMap(map);                 // 按字段值删除
boolean remove(Wrapper<User> queryWrapper);

// 查询
User getById(1L);
List<User> list();                        // 查全部
List<User> list(Wrapper<User> queryWrapper);
int count(Wrapper<User> queryWrapper);

// 更新
boolean updateById(User entity);
boolean update(User entity, Wrapper<User> updateWrapper);

✅ 2. 分页查询（直接返回 IPage）

IPage<User> page = new Page<>(1, 10);
IPage<User> result = userService.page(page, 
    new LambdaQueryWrapper<User>().ge(User::getAge, 18)
);

✅ 3. 链式查询（QueryChain / UpdateChain）

// 链式查询
List<User> list = userService.query()
    .likeRight(User::getName, "张")
    .ge(User::getAge, 18)
    .list();

// 链式更新
boolean updated = userService.update()
    .set(User::getStatus, 0)
    .lt(User::getAge, 16)
    .update();

🔸 链式操作底层仍用 Wrapper，但语法更流畅。

✅ 4. 批量操作优化

// 批量插入（自动分批，避免 OOM）
userService.saveBatch(list, 1000); // 每批 1000 条

// 批量更新（需自己实现，IService 不直接提供，但可通过 update + 循环）

✅ 5. 获取单个字段（避免查全表）

// SELECT name FROM user WHERE id = 1
String name = userService.getOne(
    new LambdaQueryWrapper<User>()
        .select(User::getName)
        .eq(User::getId, 1L),
    false // 是否抛异常（false 表示不抛，返回 null）
).getName();

四、与 BaseMapper 的区别

能力BaseMapperIService
批量插入❌（需自己写 foreach XML）✅ saveBatch()（带分批）
链式调用❌✅ query().eq().list()
Service 层语义❌（属于 DAO）✅（天然在 Service 层）
事务控制需手动加 @Transactional方法内可组合多个操作，配合 @Transactional 更安全
返回值封装基础类型更高层（如直接返回 Entity/List/IPage）

💡 最佳实践：

  • 简单单表操作 → 直接用 IService
  • 复杂 SQL（JOIN/子查询）→ 在 Mapper 写 XML，Service 调用 Mapper

五、注意事项

1. 批量操作性能

  • saveBatch() 默认每 1000 条提交一次，避免大事务。
  • 底层使用 ExecutorType.BATCH（MyBatis 批处理模式）。

2. 事务管理

IService 的方法本身不带事务，需在 Service 方法上加 @Transactional：

@Transactional
public void transfer(Long fromId, Long toId, BigDecimal amount) {
    this.updateById(new User().setId(fromId).setBalance(...));
    this.updateById(new User().setId(toId).setBalance(...));
}

3. 不要滥用

  • 多表关联、复杂聚合、报表类查询 → 仍应使用自定义 Mapper + XML
  • IService 适合单表业务逻辑

六、典型场景示例

场景：批量禁用低龄用户

@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {

    @Transactional
    public void disableMinors() {
        // 先查出所有 age < 18 的用户 ID
        List<Long> ids = this.listObjs(
            new LambdaQueryWrapper<User>()
                .lt(User::getAge, 18)
                .select(User::getId),
            User::getId
        );

        if (!ids.isEmpty()) {
            // 批量更新状态
            this.update(
                new User().setStatus(0),
                new LambdaUpdateWrapper<User>().in(User::getId, ids)
            );
        }
    }
}

✅ 总结

特性说明
定位MyBatis-Plus 的通用 Service 层封装
优势减少样板代码、支持批量、链式调用、分页一体化
适用单表 CRUD、简单条件操作
不适用多表 JOIN、复杂 SQL、存储过程
搭配继承 ServiceImpl<M, T> + 实现自定义接口

📌 记住： “Mapper 写复杂 SQL，Service 用 IService 做单表逻辑” —— 这是 MP 最高效的使用姿势。
