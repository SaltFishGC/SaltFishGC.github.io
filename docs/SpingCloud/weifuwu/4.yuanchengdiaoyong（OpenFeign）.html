远程调用

OpenFeign

OpenFeign（全称：Spring Cloud OpenFeign）是 Spring Cloud 对 Netflix Feign 的增强和封装，是一个声明式的 HTTP 客户端，用于简化微服务之间的远程调用（也即强化版http客户端）

🎯 核心目标：让远程服务调用像调用本地方法一样简单。

配置

首先导入pom依赖：

  <!--openFeign-->
  <dependency>
      <groupId>org.springframework.cloud</groupId>
      <artifactId>spring-cloud-starter-openfeign</artifactId>
  </dependency>
  <!--负载均衡器-->
  <dependency>
      <groupId>org.springframework.cloud</groupId>
      <artifactId>spring-cloud-starter-loadbalancer</artifactId>
  </dependency>

版本已由父pom的依赖管理锁定：

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-dependencies</artifactId>
    <version>${spring-cloud.version}</version>
    <type>pom</type>
    <scope>import</scope>
</dependency>

然后启动feignClient：

@SpringBootApplication
@EnableFeignClients // 启用 OpenFeign
public class OrderApplication {
    public static void main(String[] args) {
        SpringApplication.run(OrderApplication.class, args);
    }
}

使用声明调用

接下来我们就可以在需要其他服务的服务那边声明所需的服务及其方法了：

@FeignClient("item-service")
public interface ItemClient {

    @GetMapping("/items")
    List<ItemDTO> queryItemByIds(@RequestParam("ids") Collection<Long> ids);
}

这里只需要声明接口，无需实现方法。接口中的几个关键信息：

  • @FeignClient("item-service") ：声明服务名称（即yml配置文件中服务名称）
  • @GetMapping ：声明请求方式
  • @GetMapping("/items") ：声明请求路径
  • @RequestParam("ids") Collection<Long> ids ：声明请求参数
  • List<ItemDTO> ：返回值类型

完成了对需要方法的封装以后，我们到业务中使用这个封装接口进行调用：

@RequiredArgsConstructor
public class CartServiceImpl extends ServiceImpl<CartMapper, Cart> implements ICartService {
    // openfeign封装接口
    private final ItemClient itemClient;

    private void handleCartItems(List<CartVO> vos) {
        // 1.获取商品id
        Set<Long> itemIds = vos.stream().map(CartVO::getItemId).collect(Collectors.toSet());
        // 2.调用openfeign封装接口完成远程信息查询
        List<ItemDTO> items = itemClient.queryItemByIds(itemIds);
        if (CollUtils.isEmpty(items)) {
            return;
        }
        // 3.转为 id 到 item的map
        Map<Long, ItemDTO> itemMap = items.stream().collect(Collectors.toMap(ItemDTO::getId, Function.identity()));
        // 4.写入vo
        for (CartVO v : vos) {
            ItemDTO item = itemMap.get(v.getItemId());
            if (item == null) {
                continue;
            }
            v.setNewPrice(item.getPrice());
            v.setStatus(item.getStatus());
            v.setStock(item.getStock());
        }
    }

OpenFeign的连接池

OpenFeign 本身不直接提供连接池，但它底层依赖的 HTTP 客户端（如 Apache HttpClient 或 OkHttp）支持连接池。因此，OpenFeign 的“连接池”能力完全由所选的 HTTP 客户端实现。

在微服务高频调用场景下，每次 HTTP 请求都新建 TCP 连接（三次握手 + TLS 握手）会带来：

  • 高延迟
  • 高 CPU/内存开销
  • 端口耗尽（TIME_WAIT）

✅ 连接池的作用：复用已建立的连接，避免重复建连，提升性能和稳定性。

OpenFeign支持的几种连接池：

客户端是否默认连接池支持性能
Java HttpURLConnection✅ 是（默认）❌ 无连接池差
Apache HttpClient❌ 否✅ 强大连接池好
OkHttp❌ 否✅ 内置连接池优秀（推荐）

⚠️ 关键结论： 不要使用默认的 HttpURLConnection！必须显式引入 HttpClient 或 OkHttp 并启用连接池。

启用OkHttp作为连接池

首先引入pom依赖：

<!-- 使用 OkHttp 作为 Feign 的底层客户端 -->
<dependency>
    <groupId>io.github.openfeign</groupId>
    <artifactId>feign-okhttp</artifactId>
</dependency>

然后通过yml启动OkHttp（注意顶格，并非cloud的子配置）：

feign:
  okhttp:
    enabled: true # 开启OKHttp功能

OpenFeign最佳实践

  • 思路1：抽取到微服务之外的公共module
  • 思路2：每个微服务自己抽取一个module

一般考虑第一种，我们完成了公共模块后，其他模块只需要引入该模块就可以直接调用写好的接口了！

统一管理需要严格规划

现在我们开始构建这个公共模块，首先引入依赖：

<dependencies>
    <!--open feign-->
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-openfeign</artifactId>
    </dependency>
    <!-- load balancer-->
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-loadbalancer</artifactId>
    </dependency>
    <!-- swagger 注解依赖 -->
    <dependency>
        <groupId>io.swagger</groupId>
        <artifactId>swagger-annotations</artifactId>
        <version>1.6.6</version>
        <scope>compile</scope>
    </dependency>
</dependencies>

后续其他模块引入后自动继承这些依赖就可以调用接口了，其他模块不用显式引入feign依赖了（但是nacos是要的！）

然后完成对openfeign接口client的编写（记得带上dto）

最后在其他模块引入该公共依赖并直接调用对应client即可，这里需要注意我们所使用的client此时并没有作为Bean被注入Ioc容器中，因为一个模块的Spring只会扫描自己模块项目下的所有文件并导入Bean，所以这里还需要再手动声明一下扫描路径，让本项目去扫描公共模块的client文件夹：

@MapperScan("com.hmall.cart.mapper")
@SpringBootApplication
@EnableFeignClients(basePackages = "com.hmall.api.client")	// 在此处声明公共模块的client文件夹
public class CartServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(CartServiceApplication.class, args);
    }
}

日志

OpenFeign只会在FeignClient所在包的日志级别为DEBUG时，才会输出日志。而且其日志级别有4级：

  • NONE：不记录任何日志信息，这是默认值。
  • BASIC：仅记录请求的方法，URL以及响应状态码和执行时间
  • HEADERS：在BASIC的基础上，额外记录了请求和响应的头信息
  • FULL：记录所有请求和响应的明细，包括头信息、请求体、元数据。

Feign默认的日志级别就是NONE，所以默认我们看不到请求日志。

我们可以在公共feign模块中加入一个config文件来使得日志可以全局快速选择是否生效：

public class DefaultFeignClient {
    @Bean
    public Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }
}

注意是feign的Logger.Level，且不要加@Configuration注解！

然后在@EnableFeignClients注解中加入日志配置：

@MapperScan("com.hmall.cart.mapper")
@SpringBootApplication
@EnableFeignClients(basePackages = "com.hmall.api.client" , defaultConfiguration = DefaultFeignClient.class)
public class CartServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(CartServiceApplication.class, args);
    }
}

维度HTTP/1.1 + REST (JSON)gRPC
协议基础基于文本的 HTTP/1.1（可升级到 HTTP/2）强制基于 HTTP/2 二进制帧
数据格式JSON / XML（文本，可读）Protocol Buffers（Protobuf）（二进制，高效）
传输效率体积大、解析慢体积小（比 JSON 小 3~10 倍）、解析快
连接模式请求-响应（短连接）支持四种模式： • Unary（一元） • Server Streaming • Client Streaming • Bidirectional Streaming
类型安全弱类型（靠文档约定）强类型（.proto 文件生成代码，编译时检查）
跨语言通用但需手动处理官方支持 10+ 语言，自动生成客户端/服务端代码
浏览器支持原生支持❌ 不直接支持（需 gRPC-Web + Proxy）
典型场景公开 API、Web 前端调用内部微服务通信、高并发低延迟系统
  • 对外 API → 用 HTTP/REST（兼容性好）
  • 内部微服务 → 优先用 gRPC（性能高、契约强）
