IService的批量插入

SaltFishGC2026-1-29

谨记mysql批量插入尽可能使用批处理的方式而非一口气全部插入！

rewriteBatchedStatements 是 MySQL JDBC 驱动（Connector/J） 中的一个重要参数，和 Redis 的 Pipeline 有相似目标（提升批量操作性能），但实现机制完全不同。下面详细解释：

一、rewriteBatchedStatements 是什么？

✅ 定义

它是 MySQL JDBC 驱动的一个 连接参数（Connection Property），用于优化批量执行（addBatch() + executeBatch()）的性能。

默认值：false 建议生产环境设为：true

✅ 作用

当启用时，JDBC 驱动会自动将多条相同的 INSERT/UPDATE 语句重写（rewrite）成一条多值 SQL，从而减少网络往返次数和服务器解析开销。

二、工作原理（对比说明）

📌 场景：批量插入 3 条记录

PreparedStatement ps = conn.prepareStatement("INSERT INTO user(name, age) VALUES(?, ?)");
ps.setString(1, "张三"); ps.setInt(2, 20); ps.addBatch();
ps.setString(1, "李四"); ps.setInt(2, 25); ps.addBatch();
ps.setString(1, "王五"); ps.setInt(2, 30); ps.addBatch();
ps.executeBatch();
情况 1：rewriteBatchedStatements=false（默认）

驱动会发送 3 条独立 SQL 到 MySQL：

INSERT INTO user(name, age) VALUES('张三', 20);
INSERT INTO user(name, age) VALUES('李四', 25);
INSERT INTO user(name, age) VALUES('王五', 30);

→ 3 次网络往返 + 3 次 SQL 解析

情况 2：rewriteBatchedStatements=true

驱动自动重写为 1 条多值 SQL：

INSERT INTO user(name, age) VALUES('张三', 20), ('李四', 25), ('王五', 30);

→ 1 次网络往返 + 1 次 SQL 解析，性能大幅提升！

🔸 同样适用于 UPDATE（条件相同时）：

UPDATE user SET status=0 WHERE id IN (1,2,3); -- 如果原 batch 是分别 update id=1,2,3

三、如何配置？

在 JDBC URL 中添加参数：

// MySQL 5.7 / 8.0
jdbc:mysql://localhost:3306/test?rewriteBatchedStatements=true

// 更完整的生产配置（推荐）
jdbc:mysql://localhost:3306/test?
    rewriteBatchedStatements=true&
    useServerPrepStmts=false&
    allowMultiQueries=true

⚠️ 注意：

  • 必须使用 PreparedStatement + addBatch()/executeBatch()，直接拼接 SQL 字符串无效。
  • 对 Statement（非预编译）无效。

四、和 Redis Pipeline 的对比

特性MySQL rewriteBatchedStatementsRedis Pipeline
目标减少网络往返 + SQL 解析开销减少网络往返（RTT）
实现层JDBC 驱动层（客户端重写 SQL）Redis 客户端协议层（批量发送命令）
是否改变 SQL/命令✅ 重写为单条多值 SQL❌ 命令不变，只是批量发送
服务端感知服务端看到的是单条 SQL服务端依次执行多个命令
适用操作INSERT / UPDATE（同结构）任意 Redis 命令（GET/SET/HMSET 等）
事务性不保证原子性（除非外层加事务）不自动开启事务（需配合 MULTI/EXEC）

💡 相似点：

  • 都是为了 降低多次小操作的网络延迟开销
  • 都是 客户端优化技术

❌ 关键区别：

  • MySQL 的 rewriteBatchedStatements 会改写 SQL 语句本身
  • Redis Pipeline 只是“打包发送”，命令内容不变

🌰 举例：

  • Redis Pipeline：发送 [SET a 1, SET b 2, GET a] → 服务端执行 3 条命令
  • MySQL rewrite：把 3 条 INSERT → 合并成 1 条 INSERT ... VALUES (), (), ()

五、性能提升效果

根据官方和社区测试：

  • 批量插入 10,000 条记录：
    • rewriteBatchedStatements=false：可能需要 10~30 秒
    • rewriteBatchedStatements=true：通常 < 1 秒

✅ 这是 MyBatis-Plus / JPA / Hibernate 等 ORM 框架做批量插入时必须开启的参数！

同时谨记mysql批量插入尽可能使用批处理的方式而非一口气全部插入！

六、MyBatis-Plus 中如何配合使用？

MP 的 saveBatch() 底层就是用 PreparedStatement.addBatch()，所以：

1. 数据源配置（application.yml）

spring:
  datasource:
    url: jdbc:mysql://...?rewriteBatchedStatements=true&useSSL=false
    driver-class-name: com.mysql.cj.jdbc.Driver

2. 调用批量方法

List<User> list = ...; // 1000 条数据
userService.saveBatch(list, 1000); // 自动分批 + 利用 rewrite 优化

🔸 如果不加 rewriteBatchedStatements=true，saveBatch() 性能会很差！

✅ 总结

问题答案
rewriteBatchedStatements 是干什么的？MySQL JDBC 驱动参数，将批量 INSERT/UPDATE 重写为单条多值 SQL，提升性能
和 Redis Pipeline 一样吗？目标相似（减少网络开销），但机制不同：MySQL 改写 SQL，Redis 只是批量发命令
要不要开？必须开！ 尤其在使用 MyBatis-Plus saveBatch() 时
怎么开？在 JDBC URL 加 ?rewriteBatchedStatements=true

💡 最佳实践： 所有使用 MySQL 批量操作的 Java 应用，JDBC URL 务必加上 rewriteBatchedStatements=true！
