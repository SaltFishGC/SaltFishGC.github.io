IService的链式调用

SaltFishGC2026-1-29

MyBatis-Plus（MP）中的 IService 的 Lambda 链式调用 是其 Service 层最优雅、高效的编程方式之一。它结合了 Lambda 表达式（类型安全） + 链式语法（流畅 API） + 内置 CRUD 逻辑，让你在 Service 层几乎不用写重复代码。

下面从 原理、API 分类、详细用法、示例、注意事项 等方面全面讲解。

一、什么是 Lambda 链式调用？

MP 在 IService 中提供了两个核心链式入口：

链式构建器用途
query()构建 查询条件（返回 QueryChainWrapper<T>）
update()构建 更新条件（返回 UpdateChainWrapper<T>）

它们底层仍基于 LambdaQueryWrapper / LambdaUpdateWrapper，但封装成更流畅的链式 API。

✅ 特点：

  • 方法链式调用：.eq().like().list()
  • Lambda 字段引用：User::getName（避免字符串硬编码）
  • 自动执行：以 .one() / .list() / .update() 结尾即执行

二、QueryChain（查询链）

🔧 1. 入口方法

QueryChainWrapper<T> query();

🔧 2. 常用链式方法（条件）

方法说明示例
eq(Fn<S, R> column, Object val)等于.eq(User::getAge, 18)
ne(...)不等于.ne(User::getStatus, 0)
gt/lt/ge/le大于/小于等.ge(User::getScore, 90)
like/likeLeft/likeRight模糊匹配.likeRight(User::getName, "张")
isNull/isNotNull空值判断.isNotNull(User::getEmail)
in/notInIN 查询.in(User::getId, Arrays.asList(1,2,3))
between范围.between(User::getCreateTime, start, end)
orderByAsc/Desc排序.orderByDesc(User::getId)
groupBy分组.groupBy(User::getDeptId)
havingHAVING 条件.having("COUNT(*) > {0}", 5)

💡 所有字段参数使用 方法引用：Entity::getField（要求有 getter）

🔧 3. 终止方法（触发执行）

方法返回值说明
one()T返回单个对象（无结果返回 null）
one(RowBounds rowBounds)T分页取一条
list()List<T>返回列表
list(RowBounds rowBounds)List<T>分页列表
count()Long返回记录数
exists()boolean是否存在满足条件的记录

✅ 4. 查询链示例

（1）基本查询

List<User> users = userService.query()
    .likeRight(User::getName, "张")
    .ge(User::getAge, 18)
    .orderByDesc(User::getCreateTime)
    .list();

（2）查单条

User user = userService.query()
    .eq(User::getEmail, "zhang@example.com")
    .one(); // 自动 LIMIT 1

（3）是否存在

boolean exists = userService.query()
    .eq(User::getPhone, "13800138000")
    .exists();

（4）聚合统计

Long count = userService.query()
    .eq(User::getStatus, 1)
    .count();

⚠️ 注意：count() 不支持 SELECT COUNT(x) 中的字段指定，始终是 COUNT(1)。

三、UpdateChain（更新链）

🔧 1. 入口方法

UpdateChainWrapper<T> update();

🔧 2. 核心方法

方法说明
set(Fn<S, R> column, Object val)设置字段值
set(boolean condition, Fn<S, R> column, Object val)条件设置
eq/ne/gt/like...WHERE 条件（同 QueryChain）
update()执行更新，返回 boolean

✅ 3. 更新链示例

（1）简单更新

boolean updated = userService.update()
    .set(User::getStatus, 0)
    .set(User::getUpdateTime, LocalDateTime.now())
    .eq(User::getAge, 16)
    .update(); // 返回是否成功

生成 SQL：

UPDATE user 
SET status = 0, update_time = '2026-01-29 19:00:00'
WHERE age = 16;

（2）条件更新（动态）

String newName = "李四";
Integer newAge = null;

userService.update()
    .set(User::getName, newName)
    .set(newAge != null, User::getAge, newAge) // 仅当 newAge != null 时设置
    .eq(User::getId, 1001)
    .update();

（3）批量禁用

userService.update()
    .set(User::getStatus, 0)
    .lt(User::getAge, 18)
    .update();

四、高级技巧

✅ 1. 链式 + 分页

虽然链式本身不直接支持 IPage，但可结合 page() 方法：

IPage<User> page = new Page<>(1, 10);
IPage<User> result = userService.page(page, 
    new LambdaQueryWrapper<User>()
        .eq(User::getStatus, 1)
);

🔸 链式查询目前不能直接返回 IPage，需用传统 page(Page, Wrapper)。

✅ 2. 字段选择（避免查全表）

链式查询不支持 select() 指定字段！ 如需只查部分字段，必须回退到 Wrapper 或 XML：

// ❌ 链式不支持 select
// userService.query().select(User::getId, User::getName)... 

// ✅ 正确做法：用 LambdaQueryWrapper
List<User> list = userService.list(
    new LambdaQueryWrapper<User>()
        .select(User::getId, User::getName)
        .eq(User::getStatus, 1)
);

✅ 3. 与事务结合

链式操作本身无事务，需手动加 @Transactional：

@Transactional
public void transfer(Long fromId, Long toId) {
    boolean ok1 = this.update()
        .set(User::getBalance, balance - 100)
        .eq(User::getId, fromId)
        .update();

    boolean ok2 = this.update()
        .set(User::getBalance, balance + 100)
        .eq(User::getId, toId)
        .update();

    if (!ok1 || !ok2) throw new RuntimeException("转账失败");
}

✅4. 使用condition设置条件

所有条件方法都提供 重载版本，第一个参数为 boolean condition：

.eq(boolean condition, SFunction<T, ?> column, Object val)
.like(boolean condition, SFunction<T, ?> column, Object val)
.ge(boolean condition, SFunction<T, ?> column, Object val)
// ... 其他方法同理

🔸 只有当 condition == true 时，该条件才会被加入 SQL。

如：

public List<User> searchUsers(String name, Integer minAge, Integer maxAge, Integer status) {
    return this.query()
        // 名字模糊匹配（仅当 name 非空）
        .like(StringUtils.isNotBlank(name), User::getName, name)
        
        // 最小年龄（仅当 minAge 不为 null）
        .ge(minAge != null, User::getAge, minAge)
        
        // 最大年龄（仅当 maxAge 不为 null）
        .le(maxAge != null, User::getAge, maxAge)
        
        // 状态（仅当 status 有效）
        .eq(status != null && status >= 0, User::getStatus, status)
        
        .list();
}

五、注意事项 & 限制

问题说明
不支持多表 JOIN链式仅适用于单表，多表请用 XML
不支持自定义 SELECT 字段无法像 Wrapper 那样 .select(User::getName)
不支持复杂嵌套条件如 (A AND B) OR (C AND D)，需用 and(i -> ...).or(j -> ...)，但链式不暴露此 API（需回退到 Wrapper）
性能底层仍是 MP 的通用方法，性能与直接调用 list(wrapper) 相同
返回值update() 返回 boolean（是否影响行数 > 0），不是影响行数

六、完整示例：用户管理场景

@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {

    // 启用用户
    public boolean enableUser(Long id) {
        return this.update()
            .set(User::getStatus, 1)
            .set(User::getUpdateTime, LocalDateTime.now())
            .eq(User::getId, id)
            .eq(User::getStatus, 0) // 确保之前是禁用状态
            .update();
    }

    // 搜索活跃用户
    public List<User> searchActiveUsers(String keyword) {
        return this.query()
            .like(StringUtils.isNotBlank(keyword), User::getName, keyword)
            .eq(User::getStatus, 1)
            .ge(User::getLastLoginTime, LocalDateTime.now().minusDays(30))
            .orderByDesc(User::getLastLoginTime)
            .list();
    }

    // 检查邮箱是否已注册
    public boolean isEmailExists(String email) {
        return this.query()
            .eq(User::getEmail, email)
            .exists();
    }
}

✅ 总结：何时用链式调用？

场景推荐
单表简单 CRUD✅ 强烈推荐（代码简洁）
动态条件查询✅（配合 boolean 条件）
需要指定查询字段❌ 用 LambdaQueryWrapper
多表关联 / 复杂 SQL❌ 用 XML
需要分页返回 IPage❌ 用 page(Page, Wrapper)

💡 口诀： “单表操作用链式，字段选择用 Wrapper，多表复杂写 XML。”

通过 IService 的 Lambda 链式调用，你可以写出类型安全、可读性强、零 SQL 字符串的业务代码，大幅提升开发体验和维护性。
