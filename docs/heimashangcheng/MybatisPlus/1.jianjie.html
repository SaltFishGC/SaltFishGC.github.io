MybatisPlus简介

SaltFishGC2026-1-29

mybatis只是个方便写sql的框架，只是映射了Entity，方便获取返回ResultSet。

而mybatisplus是基于mybatis的ORM框架（不用写sql，至少单表不用）

现在简要介绍一下：

MyBatis-Plus（简称 MP）是 MyBatis 的增强工具，在 MyBatis 的基础上只做增强，不做改变，为简化开发、提高效率而生。以下是 从配置到应用 的简要介绍：

一、核心特性

  • 无侵入：基于 MyBatis，不改变其原有功能。
  • 强大的 CRUD 封装：内置通用 Mapper、Service，无需写 SQL 即可完成基本操作。
  • Lambda 表达式支持：避免硬编码字段名。
  • 自动分页、逻辑删除、乐观锁、代码生成器等。

二、基本配置（以 Spring Boot 为例）

1. 引入依赖（Maven）

<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-boot-starter</artifactId>
    <version>3.5.7</version> <!-- 使用最新稳定版 -->
</dependency>
<!-- 数据库驱动，如 MySQL -->
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
</dependency>

2. 配置数据源（application.yml）

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/test?useUnicode=true&characterEncoding=utf8
    username: root
    password: root
    driver-class-name: com.mysql.cj.jdbc.Driver

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
  global-config:
    db-config:
      id-type: auto  # 主键策略

3. 启动类添加 @MapperScan

@SpringBootApplication
@MapperScan("com.example.mapper") // 扫描 Mapper 接口
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

三、实体类与 Mapper 定义

1. 实体类（Entity）

@Data
@TableName("user") // 指定表名（若类名与表名一致可省略）
public class User {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String name;
    private Integer age;
    private String email;
}

2. Mapper 接口

public interface UserMapper extends BaseMapper<User> {
    // 继承 BaseMapper 即拥有 17+ 个通用方法（insert, selectById, update 等）
}

四、基本使用示例

1. 增删改查（无需写 SQL）

@Autowired
private UserMapper userMapper;

// 插入
userMapper.insert(new User().setName("张三").setAge(20));

// 查询
User user = userMapper.selectById(1L);

// 更新
User update = new User();
update.setId(1L);
update.setName("李四");
userMapper.updateById(update);

// 删除
userMapper.deleteById(1L);

2. 条件构造器（Wrapper）

// 查询 name 以 '张' 开头且 age > 18 的用户
QueryWrapper<User> wrapper = new QueryWrapper<>();
wrapper.likeRight("name", "张").gt("age", 18);
List<User> users = userMapper.selectList(wrapper);

// Lambda 写法（类型安全）
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<>();
lqw.likeRight(User::getName, "张").gt(User::getAge, 18);

3. 分页查询

// 配置分页插件（Spring Boot 3.x）
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
    return interceptor;
}

// 使用
Page<User> page = userMapper.selectPage(new Page<>(1, 10), null);

五、其他常用功能

  • 逻辑删除：通过 @TableLogic 注解实现软删除。
  • 自动填充：创建时间/更新时间自动注入（@TableField(fill = FieldFill.INSERT)）。
  • 代码生成器：一键生成 Entity、Mapper、Service、Controller。
  • 多租户、SQL 注入器、性能分析插件（开发环境）等。

六、复用Mapper文件

  • MP 的 BaseMapper 方法（如 selectById）默认不生成 XML，而是通过 MyBatis 的 @Select 注解或内部注入实现。
  • 你自己的 XML 中定义的 <select id="xxx"> 只要 id 不与 BaseMapper 的方法名重复，就不会冲突。
  • 即使重复，MyBatis 会优先使用 XML 中定义的 SQL（覆盖 MP 默认行为），但一般不建议这样做，除非你明确要重写。

📌 BaseMapper 的方法名包括： insert, deleteById, updateById, selectById, selectList, selectPage 等。 只要你的 XML 中不要写同名的 <select id="selectById">（除非你想覆盖），就安全。

配置步骤（Spring Boot）

1. 保留原有 XML 文件位置

例如：

src/main/resources/mapper/UserMapper.xml

2. 配置 MyBatis-Plus 扫描 XML

在 application.yml 中指定 XML 路径：

mybatis-plus:
  mapper-locations: classpath*:mapper/**/*.xml  # 扫描所有 mapper 目录下的 XML
  configuration:
    map-underscore-to-camel-case: true
  type-aliases-package: com.example.entity

⚠️ 注意：用的是 mybatis-plus.mapper-locations，不是 mybatis.mapper-locations（虽然 MP 也兼容后者，但推荐用 MP 自己的配置项）。

3. Mapper 接口继承 BaseMapper 并保留自定义方法

public interface UserMapper extends BaseMapper<User> {

    // 复用 XML 中的自定义 SQL
    List<User> selectByName(@Param("name") String name);

    // 也可以写注解方式，但 XML 更灵活
}

对应的 UserMapper.xml：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.mapper.UserMapper">

    <!-- 自定义方法，id 必须和接口方法名一致 -->
    <select id="selectByName" resultType="com.example.entity.User">
        SELECT * FROM user WHERE name LIKE CONCAT(#{name}, '%')
    </select>

    <!-- ❌ 不要随便写 selectById，除非你想覆盖 MP 默认行为 -->
    <!-- <select id="selectById">... </select> -->

</mapper>

4. 启动类确保扫描 Mapper 接口

@SpringBootApplication
@MapperScan("com.example.mapper") // 扫描你的 Mapper 接口包
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
